### Bug Fixes

**Fixed startup crash affecting multiple entities**
- Resolved `ValueError` crash in Last Cover Action sensor that prevented integration from loading
- Fixed cascading failures preventing switch entities from completing setup
- Removed incorrect empty unit attribute from text sensor that caused Home Assistant to interpret it as numeric

### Technical Details

**Root Cause:**
The Last Cover Action diagnostic sensor was configured with an empty unit of measurement (`""`), which Home Assistant's sensor platform interpreted as indicating a numeric sensor. When the sensor returned text values like "No action recorded", HA attempted to convert the text to a float, causing a `ValueError`.

**Impact:**
This crash occurred during coordinator refresh, which is triggered when switch entities complete their setup. The failure prevented:
- Last Cover Action sensor from loading
- All switch entities (automatic control, manual override, return to default, climate mode) from completing setup

**Solution:**
Text sensors without a device_class or state_class should not set a unit of measurement. The fix removes the empty unit attribute, allowing Home Assistant to correctly handle the text values.
